Net Bridge Protocol
John Haugeland, Jeff Katz$Revision$ The Net Bridge Protocol is an extremely simple reference layer for other protocols to be built on top of; its purpose is to bury the annoying details of network behavior in most languages. By design it is simple, message oriented, language portable and robust. The Net Bridge Protocol makes no attempt to handle complex issues like quality of service; it is merely a lightweight wrapper to allow a client and/or server implementation to be cranked out simply and re-usably in any given language. Definition The Net Bridge Protocol, hereafter NBP, is a simple message oriented protocol. The intent is to provide a simple string interface between languages. Except during the connection phase, which has special rules, messages are simply length prepended strings which are handed back and forth. Length prepends are unsigned 32-bit integers. The NBP is an octet oriented protocol; it may only transfer sequential strings of 8-bit bytes. These bytes may be interpreted by the implementors of the protocol however desired, but within the discussion of the protocol, they will be discussed from the C perspective of unsigned characters on an 8-bit character machine. The Net Bridge Protocol has two phases: connection phase and transit phase. The connection phase is a simple bidirectional negotiation. The transit phase is a message transfer phase. After a socket is accepted, the connection phase begins immediately. The connection phase will then either make a demand or switch to the transit phase. Demands all lead to the transit phase, more demands or a new connection; eventually the transit phase will be reached. The transit phase is then permanent until disconnection.
